# Skills under .claude/skills/

This directory holds two families of skills used by the cflx (Conflux) workflow.
Both families are **tracked operational config**, not user-local preferences.
A fresh checkout needs all of them for `cflx run` to function as configured in `.cflx.jsonc`.

## Hand-authored (cairn-specific)

| Skill | Purpose |
|---|---|
| `cflx-proposal/` | Interactive proposal authoring for new changes. |
| `cflx-run/` | Standard cflx orchestration entry. |
| `cflx-workflow/` | apply / accept / archive operations invoked by cflx-run. |

These evolve alongside CAIRN's phase conventions. Edit them when the workflow changes.

## Vendored (generated by openspec CLI)

| Skill | Generator |
|---|---|
| `openspec-apply-change/` | openspec CLI 1.3.1 |
| `openspec-archive-change/` | openspec CLI 1.3.1 |
| `openspec-explore/` | openspec CLI 1.3.1 |
| `openspec-propose/` | openspec CLI 1.3.1 |

These are emitted by the `openspec` CLI. **Do not hand-edit.**
Regenerate with the OpenSpec CLI when you upgrade. If you need to override
behavior, do it in a hand-authored cflx-* skill that calls the openspec skill,
not by patching the vendored file.

## Boundary policy

`.claude/` in this repo is a tracked operational surface:

- `.claude/settings.json` and `.claude/settings.local.json` are tracked (project-level Claude Code config).
- `.claude/skills/` and `.claude/commands/` are tracked (operational workflow assets).
- `.claude/references/` and `.claude/stronghold/` are gitignored (session-local scratch).

If you find yourself editing skills under `~/.claude/` (your user-global directory) for
work that should apply to all CAIRN contributors, move it into this repo's `.claude/skills/`
instead. Personal preferences belong in `~/.claude/`; project workflow belongs here.
