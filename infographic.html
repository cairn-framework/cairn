CAIRN

The Map Between Plans and Code

A shared architectural memory that both humans and AI read from and write to. It connects your intentions to your implementation through a permanent, structured map of your entire project.

Core Concept

Two Chains, One Hinge

Every piece of knowledge flows through two linked chains. Evidence flows up through the Provenance Chain. Rules flow down through the Authority Chain. They meet at Decisions, the only artefact with obligations in both directions.

Provenance Chain -- traces reasoning

Where did this decision come from? Sources are raw materials (documents, transcripts, links). Research synthesises those sources into findings. Decisions cite that research as justification. If the trail breaks, CAIRN surfaces it as a warning -- not a blocker, because human judgement is needed here.

Authority Chain -- enforces structure

Is the code following the plan? Decisions declare which parts of the DSL map they constrain. The DSL is the current architectural truth. Contracts spell out what each module promises. Code is what actually exists. Each link is mechanically checked -- violations block commits.

The Fourth Dimension

Changes Over Time

Changes are never applied directly to the main tree. They are proposed, worked on in isolation, and archived atomically -- creating a full audit trail of how the project evolved and why.

01

Propose

Describe what you want to change. Each change gets its own isolated directory with delta files that mark what will be added, modified, removed, or renamed.

02

Isolate

Work inside the change directory without touching the main tree. Multiple changes can be in flight at once. The main map stays clean and stable.

03

Archive

Merge the change atomically into the main tree. CAIRN processes deltas in strict order -- renamed, removed, modified, added -- to guarantee consistency.

04

Record

Every reconciliation event is appended to an immutable log. The full history of the project's structural evolution is preserved and queryable.
