# Voice and audience

Load this file when: writing user-facing copy, spec prose, marketing text, or any
string that reaches a reader.

## Audience

CAIRN's audience is broadening from career developers to "people building with AI
tools," including non-devs. User-facing vocabulary should prefer plain concise English.
Accuracy is the floor: do not flatten load-bearing technical taxonomy. The bar is
"would a non-dev feel nervous typing this command or reading this doc?", not
"what is the simplest possible word."

## Em-dash ban

Em-dashes are banned in all user-facing copy. Replace with a period, colon,
comma, or parenthesis as context dictates. Applies to spec prose, marketing copy,
UI strings, and every file in this repo.
