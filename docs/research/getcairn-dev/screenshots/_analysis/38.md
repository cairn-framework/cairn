## Summary
This screenshot shows an interactive block-diagram canvas representing a marine robotics system composed of six subsystems linked by interface connections. Six subsystem cards (USV Platform, Power Generation and Distribution, ROV Vehicle, Launch and Recovery System, Communications and Data Link, Autonomy Control and Mission Payload) are arranged in two rows with orthogonal connector lines and small port nodes joining them. A floating toolbar in the upper right offers zoom and view controls, and a minimap in the lower right previews the layout. Below the canvas, an Interface Control Document panel lists the two visible connections with protocol and signal metadata. The scene presents CAIRN's contract or interface graph as a draggable systems-engineering schematic rather than a textual artefact list.

## Novel observations
- First view in the set showing a true draggable systems-engineering canvas with orthogonal routing and port nodes, rather than a static artefact list or static graph render.
- Introduces the term "INTERFACE CONTROL DOCUMENT" (ICD), a portable systems-engineering vocabulary borrowed from aerospace and marine engineering and surfaced here as a tabular companion to the visual graph.
- Subsystem typography uses uppercase tag plus serif title plus monospaced id triplet, consistent with the design system's serif-plus-mono pairing.
- Protocol values render as inline pill chips with multi-line content (e.g. "HVDC power + fibre-optic data"), a new presentation pattern for typed-edge metadata.
- The minimap and view-mode toolbar imply a multi-mode canvas (likely list, grid, graph), not previously seen.
- Subsystem ids follow a "SUB-<TAG>" convention distinct from the artefact id conventions seen earlier (e.g. CON-, DEC-).
- Domain example shifts from software/CAIRN-self to a marine robotics USV/ROV scenario, suggesting CAIRN positioning beyond pure code into hardware systems.
