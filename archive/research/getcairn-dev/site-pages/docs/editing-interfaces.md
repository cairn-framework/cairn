## Overview

[Cairn](/) is an AI-native model-based systems engineering platform that transforms informal design conversations into persistent, inspectable engineering models rather than disposable chat transcripts.

## Core Concept

The fundamental distinction: "AI should contribute structure to a persistent, inspectable engineering model, not generate disposable text that vanishes when the chat window closes."

## Workflow: Four Steps

**Describe**. Users input rough system ideas in plain language.

**Decompose**. AI synthesizes the description into explicit subsystems, components, requirements, and interfaces with persistent IDs.

**Inspect**. Twelve analytical lenses (Overview, Requirements, Architecture, Visuals, Verification, etc.) view the same underlying model without context switching.

**Refine**. AI proposes structural changes as reviewable ChangeSets; users accept or skip individual modifications.

## Key Features

- **Persistent Models**: System trees, requirements, interfaces, and verification evidence remain editable and traceable
- **Multiple Visualization Lenses**: Requirements views, architectural diagrams, 2D/3D renders, simulation results, and causal analysis
- **Full Traceability**: Every requirement links to implementing components and verification results
- **AI-Assisted Refinement**: Proposals arrive as discrete, reviewable decisions
- **Integrated Assets**: 2D concept renders, 3D meshes, and Monte Carlo simulations stay tied to model nodes

## What Persists vs. Vanishes

**Persists**: System trees, requirements, interfaces, verification data, change history, evidence

**Vanishes**: Chat transcripts, temporary markdown, context window state, prompt scaffolding

## Try It

Users can describe any system, rovers, satellites, drones, enclosures, without account setup.
