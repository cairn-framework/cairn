# Lenses Documentation

**Source:** https://www.getcairn.dev/docs/lenses
**Captured:** 2026-04-28

## Overview

Cairn offers 12 distinct lenses that each present a unique engineering perspective on selected nodes within a system design.

## The 12 Lenses

**Overview**
A comprehensive dashboard displaying node information, health metrics, budget allocations, and requirement summaries.

**Brief**
"AI-generated narrative documentation with 8 sections covering purpose, capabilities, operating modes, architecture, interfaces, constraints, verification, and risks."

**Visuals**
Two-dimensional concept renderings created through image generation alongside three-dimensional mesh assets built with Three.js technology.

**Requirements**
Functional, performance, interface, safety, environmental, and constraint requirements with inline editing capabilities and AI-powered generation features.

**Architecture**
An interactive node graph visualization depicting subsystems, interfaces, and data flow patterns with drag-and-drop layout functionality.

**Causality**
A pyramid framework showing prerequisite technologies organized from foundational knowledge through domain-specific technologies, color-coded by maturity level.

**Completeness**
A three-axis fidelity radar assessing entity, process, and relationship coverage per node with system-wide heatmap visualization.

**Narrative**
A systemigram visualization identifying transformation chains and rendering readable narrative diagrams with navigation capabilities.

**Dendritic**
An alternatives tree displaying active and rejected design paths with engineering rationales and evaluation histories.

**Behavior**
State machines illustrating states, transitions, guards, and actions with visual state diagrams.

**Verification**
Test records, analysis results, and inspection documentation linked to requirements with coverage tracking.

**Operational**
Use cases and operational scenarios detailing actors, preconditions, postconditions, and step sequences.
