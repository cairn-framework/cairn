# Two-Tier Export: Local Quick vs AI-Powered

**Source:** _resized/64-settings-export-options.png
**Captured:** 2026-04-28 (getcairn.dev web UI, Settings → Export)

## Summary
The Export Project panel splits into two tiers rather than offering a single configurable dialog. The top tier ("Quick Export (Local)") is gated by no API key and yields three instant downloads: JSON, Markdown, and a "Requirements CSV." The bottom tier ("Professional Export (AI-Powered)") requires an API key and produces PPTX presentations or DOCX reports, saved to "your Assets" rather than downloaded directly. There are no scope selectors, no include/exclude toggles, no field pickers, and no disclaimers about what is or is not included. The sharing model assumes the *whole project* is the export unit, with format being the only choice.

## Novel observations
- **Two-axis taxonomy of export.** The split is not by format but by *where the work happens* (local vs API-powered) and *where the result lands* (download vs Assets folder). Cairn's framework currently thinks of export as a single "render the graph" operation; getcairn separates "dump the data" from "compose a deliverable."
- **"Requirements CSV" is a domain-specific shape**, not a generic table export. The label assumes the project's content is requirements, suggesting a PM/spec-document mental model rather than a knowledge-graph model.
- **Assets is treated as a first-class destination.** "Saved to your Assets" implies a project-internal asset library distinct from the user's filesystem. Worth noting for our own thinking about where generated artefacts live (in-graph vs side-channel).
- **Latency is surfaced as a UI element** ("~30s") next to the cost gate ("Requires API key"). Honest tradeoff display, useful pattern.
- **Zero scope/filter controls.** Operationally this means users are expected to export everything, every time. No partial-graph share, no "this neighbourhood only," no redaction. The sharing model assumes the project is small enough or trusted enough to ship whole.
- **No format negotiation between tiers.** You cannot get a Markdown report that is AI-composed, nor a JSON that is "professionally" enriched. Format is hard-bound to tier.
- **Portable to CAIRN:** the local-vs-AI split could map to "raw graph export" (JSON, map.md, contract bundles) versus "narrative artefacts" (decision rationales rendered to DOCX, blueprint walkthroughs to PPTX). The asset-library destination is a useful pattern when generated documents should remain inside the provenance chain rather than escape to the user's filesystem.
- **Missing distinct terms:** "Quick" vs "Professional" framing is marketing language layered onto a technical distinction (sync local vs async LLM). CAIRN should pick clearer terms.
