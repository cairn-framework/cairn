# Settings Appearance and AI Configuration

**Source:** _resized/53.png
**Captured:** 2026-04-28 (getcairn.dev web UI)

## Summary
A settings page on a light cream background showing three stacked sections: Appearance, AI Configuration, and the start of Image AI Configuration. Appearance offers a three-way Theme toggle (Light, Dark, System) with Light currently active. AI Configuration provides an Anthropic API key input with a "Show" reveal control, a green status pill confirming a managed API fallback, and a Model selector currently fixed on Claude Sonnet 4. The bottom of the screen begins an Image AI Configuration section keyed by Gemini.

## Novel observations
- Distinct "managed API" concept: Cairn fronts model access by default, with personal quota as an opt-in. Phrasing pattern reusable for any hosted-AI app with BYOK.
- Two-provider split: text generation (Anthropic) and image generation (Gemini) configured in separate sections, signalling provider specialisation per modality.
- Locked Model selector with forward-looking copy ("Future versions will support model selection.") is a transparent pattern for shipping a single-model release while reserving design space.
- Status pill uses a coloured dot plus inline guidance, combining badge and tooltip into one row.
- Mono-font model chip with a leading diamond glyph treats the model name as a code-like token, distinct from sans UI chrome.
- Three-way theme toggle uses celestial glyphs (sun, crescent, half-moon) to encode Light, Dark, System without relying on text alone.
