## Novel observations
- "Dendrite" is presented as a top-level workspace tab alongside Overview, Brief, Visuals, Requirements, Architecture, Causality, Completeness, Narrative, Verification: this is the canonical project navigation and likely portable to other artefact-tree tools.
- Counters above the tree use a tri-fold rubric: "active", "pruned", "depth N", combining graph-pruning state with a depth filter in one strip.
- "Pinned (N)" pill sits beside the search box, indicating the dendrite supports node-pinning for persistent context.
- Tag chips combine semantic class (ENGINEERING, DECISION) with topology hints (ACTIVE PATH, DEPTH 1), mixing classification and graph-position metadata.
- Each tree node carries its tag row inline (ENGINEERING DECISION etc.), so taxonomy is visible at every level rather than only on selection.
- The detail panel echoes the description twice: once in the header subtitle and again under a "DESCRIPTION" section, hinting at a header-summary plus full-body convention with Show more truncation.
- Domain example is engineering-decomposition (USV-ROV survey vessel), reinforcing that CAIRN is positioned beyond software for general systems decomposition.
