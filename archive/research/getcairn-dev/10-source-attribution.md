## 3. Export-derived facts

The following facts came directly from the offshore-survey project export JSON (`offshore-survey-usv-rov-0.1.0.cairn/`) and the settings export markdown, not from screenshots. The user can re-verify these by re-running the export.

**Manifest (`manifest.json`):**

- Export envelope version `1.0`, exported at `2026-04-28T13:54:40.587Z`.
- Project name `Offshore Survey USV-ROV`, project version `0.1.0`.
- Asset count 2; total asset size 943,436 bytes; model size 198,039 bytes.
- The `.cairn` extension is a directory bundle, not a single file.

**Project schema (`project.json`):**

- Format version `1`, schema version `1.0.0`.
- Root has 19 keys; 7 are object-shaped metadata, 12 are top-level arrays of records.
- 7 nodes total (1 system, 6 subsystems); decomposition depth 2.
- 9 requirements, 1 verification, 2 interfaces, 0 trace links populated, 1 data type, 6 units, 0 use cases, 0 states, 0 transitions, 0 simulations.
- 2 assets in the registry (1 image, 1 mesh script), 1 mesh asset, 1 visual meta, 1 visual series.
- 7 history changesets recorded.

**Node taxonomy:**

- Type field carries values `system` (1) and `subsystem` (6); no `module`, `actor`, `external_system`, or `container` types in this export.
- ID prefix scheme observed: `SYS-<slug>` for system, `SUB-<name>` for subsystem; `REQ-NNN` for requirements, `IF-NNN` for interfaces, `VER-<hash>` for verifications, `DT-NNN` for data types, `UNIT-NNN` for units, `ast-<hash>` for assets, `mesh-<hash>` for meshes, `vis-<hash>` for visuals, `vs-<hash>` for visual series.
- Subsystem ids in this export use `SUB-<name>` and the system uses `SYS-<slug>`, both outside the `project.settings.namingPrefixes` table (`REQ-`, `IF-`, `S-`, `T-`, `VER-`, `UC-`).

**Requirements:**

- Twelve fields per record: `id, nodeId, parentReqId, title, description, type, rationale, acceptanceCriteria[], priority, sortOrder, createdAt, createdBy`.
- All nine requirements have priority `must` and `createdBy: ai`.
- Type distribution: 5 `performance`, 2 `functional`, 1 `environmental`, 1 `safety`.
- REQ-007, REQ-008, REQ-009 derive from REQ-003 via `parentReqId`.

**Verifications:**

- Single record (`VER-fp596o`) with method `test`, status `draft`, results empty string, created by `user`.
- No structured evidence linkage in the schema (no `evidenceAssetId` field; `results` is plain text).

**Interfaces:**

- Two interfaces, eight signals total.
- `IF-001` Power Generation → ROV Vehicle, protocol `HVDC power + fibre-optic data`, 4 signals.
- `IF-002` Autonomy → Communications, protocol `Ethernet / IP`, 4 signals.
- All 8 signals reference `DT-001` (the only data type defined).
- `interfaceHash` field exists only inside `_narrativeAnalysis` and holds the value `"IF-001|IF-002"` (a pipe-joined id list, not a content hash).

**`_narrativeAnalysis` block:**

- Attached only to the system root node.
- Structure: `{ generatedAt, interfaceHash, mainstaySentence, cards[], connections[], islands[], layout, mainstayPath }`.
- `mainstaySentence`: a single English sentence summarising the canonical causal chain across subsystems.
- `connections[].verbPhrase`: generated English (e.g. "energises and commands"); `role` enum observed: `mainstay`.
- The block persists across both project and settings exports.

**History and pipeline trace:**

- 7 changesets total. Operations: 25 `create`, 3 `update`. Collections touched: nodes (9), requirements (9), units (6), interfaces (2), data types (1), verifications (1).
- Specialists observed: `architect` (1), `brief` (2), `requirements` (2). Two changesets have `specialist: null` and `author: user`.
- Prompt versions observed: `architect-v1.0`, `brief-v1.0`, `requirements-v1.0`, `requirements-LARS-v1.0`.
- `pipelineTrace.stages[]` captures `router → context → specialist → validator` for all five AI changesets.
- Models observed in pipeline: `claude-haiku-4-5-20251001` for router, `claude-sonnet-4-6` for specialist, null model for context and validator (deterministic stages).

**Genesis (inception interview):**

- 3 rounds, 12 questions total in this export.
- `confidence` ranges 0..1; final value 0.82.
- `buildResults` records subsystemCount, interfaceCount, totalNodeCount, briefSectionCount, buildDurationMs.

**Usage telemetry (`usage.json`):**

- 26 metering records; total 162,639 tokens; total estimated cost ~$1.13.
- Category split: `inception` (4 calls, $0.12), `pipeline` (18 calls, $0.71), `visual` (2 calls, $0.04), `mesh` (2 calls, $0.25).
- Provider split: 25 Anthropic calls plus 1 Gemini call (image generation).
- Models observed: `claude-sonnet-4-6` (specialists, mesh, narrative, brief), `claude-haiku-4-5-20251001` (routers, visual digest), `gemini-3.1-flash-image-preview` (visual).

**Project versus settings export diff:**

- Settings export omits `history[]` (7 changesets including `pipelineTrace`, `prompt`, `promptVersion`, `specialist`, `proposedBy`, `approvedBy`).
- Settings export omits binary asset payloads; project export ships them in `assets/`.
- Settings export adds a single `_exportNote` advisory string and sibling pretty-printed views (`*.md`, `*.csv`).
- Sampled REQ-001 is bit-for-bit identical in both exports.

**Markdown render mapping:**

- The settings export's MD render uses the system node's `description` (not `properties.brief`) for the System Overview.
- Subsystems are rendered in `sortOrder` order with `name`, `type` and `id` plus a per-subsystem Interfaces table when the subsystem is an interface endpoint.
- Requirements grouped by node name (not id), Type and Priority columns only.
- Verification table joins `verifications[]` to `requirements[]` for the title prefix.
- Suppressed in the MD render: `description`, `rationale`, `acceptanceCriteria`, `parentReqId`, all `params[]`, all `brief.sections[]`, the entire `_narrativeAnalysis`, signal `dataTypeId` and `rate`, all asset/mesh/visual metadata, the genesis interview transcript, and any `traceLinks` (none populated in this export).

**CSV render mapping:**

- 9 data rows, 7 columns (`ID, Title, Description, Type, Priority, Node, Verification Status`).
- `Node` column resolves the requirement's `nodeId` to the human-readable node name.
- `Verification Status` aggregates `verifications[]` filtered by `requirementId`. REQ-001 shows `draft`; all other requirements show `none`.
- Omits `rationale`, `acceptanceCriteria`, `parentReqId`, `sortOrder`, `createdAt`, `createdBy`. The CSV is a review-grade summary, not a round-trip artefact.
